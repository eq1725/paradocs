### Recurring patterns and lessons from this session

- **Screenshot-driven iteration.** Chase posts a screenshot, we panel-review, ship. Don't trust the in-context "this looks fine" — get the live render then iterate.
- **Layout restructures need explicit typecheck before commit.** V10.7.B.4 shipped a JSX-imbalance bug (unclosed `<div>`) because I trusted a stale typecheck. Always run `npx --no-install tsc --noEmit -p tsconfig.json 2>&1 | grep -E "<file>" | head` AFTER editing layout JSX.
- **Centerline alignment in CSS grid.** When labels and values have different content heights, `items-center` not `items-start`. We hit this in V10.7.B.8 on the dateline grid.
- **Chip padding offsets text from grid column boundary.** Plain text + `·` separators reads more aligned than chip pills when sharing a grid column. We dropped chip pills in V10.7.B.6 for this reason.
- **Premature engagement asks reduce conversion.** Putting a resonance button before the facts (title → answer → resonance → meta) is worse than after the facts (title → answer → meta → resonance). UX journey matters.
- **Claim-check false positives are a real problem.** Strict claim-check rejected good narratives in the Kansas case. V10.7.D loosened it — verify before assuming fixed.
- **`PROMPT_VERSION` constant matters.** Bump it whenever you change any prompt in the rewrite-pipeline or downstream services so audit rows are filterable.
