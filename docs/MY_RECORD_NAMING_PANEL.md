# Naming the Spine — Go-to-Market Naming Panel

_What do we call the personal membership surface currently working-titled "My Record"? Panel of market-research, product-marketing, mass-market, and conversion/engagement voices proposes; a red team attacks; we keep what survives._

---

## 0. The brief the name must serve

The surface is: **your one strange experience, held with dignity, connected to everyone who shares it, and changing a little every time you return.** The name greets a **mass-market newcomer** (not an enthusiast), at the emotional peak right after they've shared something they may never have told anyone. It must read warm and human, never clinical or admin-y, and it carries the **possessive** (it's *theirs*).

**Evaluation criteria:** (1) instant clarity to a normal person, (2) warmth / dignity, (3) ownership pull (the IKEA/identity effect that drives return + conversion), (4) differentiation/brand fit, (5) works as a nav label *and* in a sentence ("Open your ___"), (6) doesn't over-promise or trigger safety/credibility baggage.

---

## 1. PANEL 1 — candidates and the case for each

*Voices: Brand Strategist, Mass-Market Product Marketer, Conversion/Engagement Lead, Consumer Researcher, Domain Expert.*

| Candidate | The pitch | Register |
|---|---|---|
| **My Record** (incumbent) | Clear, documentary, already in code. "Record" = both *the account you recorded* and *a thing of permanence* (a record that stands). | Neutral / slightly clerical |
| **Your Record** | Same, but second-person is warmer and more inviting at the reveal moment ("This is *your* Record"). | Warm, neutral |
| **My Constellation** | On-brand (the product already plots kindred as a night-sky/constellation; RADAR, ConstellationReveal). Your experience = one star among the others who share it. Belonging baked in. | Evocative, branded |
| **My Place** / **My Place in the Archive** | Belonging-first. "Where I fit." Humble, human, ownership-heavy. | Warm, plain |
| **Echoes** / **My Echoes** | The core emotional payoff ("others echoed this") *is* the name. Poetic, memorable. | Evocative |
| **Kindred** | Names the differentiator (people like you). Strong belonging. | Evocative, slightly literary |
| **My Case** / **Casefile** | Investigative, documentary, on-genre (X-Files energy). | Genre-y, can feel cold |
| **My Signal** | "You're not noise — you're a signal." On-brand with the matching engine. | Abstract |

**Marketer's lean:** the winner pairs **possessive warmth** with **belonging**. "My Record" is *clear* but clerical; the most *converting* names make the user feel *seen and connected*, which is what this surface sells. Top of the marketer's list: **Your Record**, **My Constellation**, **My Place**.

**Researcher's flag:** test with the wedge segment ("I had an experience I never told anyone"). For them, "Record" can read bureaucratic (a record = a file the system keeps on you); a warmer/belonging name likely tests better on *desire to return*. But "Record" wins on *instant comprehension* — no one is confused by it.

---

## 2. PANEL 2 — the red team

**Attack 1 — "Evocative names fail the comprehension test."** "Echoes," "Kindred," "Signal," "Constellation" are beautiful but a mass-market newcomer may not know *what the tab does*. Nav labels must be legible in <1 second. A clever name that needs explaining loses the casual user.
**Fix:** Reserve evocative words for *headlines inside* the surface ("Your experience echoed by 47 others"), and keep the *nav label* legible. The connective tissue ("Constellation," "Echoes," "Kindred") can be the **section/feature** name, not the whole-surface name.

**Attack 2 — "'Record' carries a surveillance connotation."** "A record" can read as *a file the institution keeps on you* — the opposite of the warm, user-owned framing. For a trust-sensitive, sometimes-vulnerable audience, that's a subtle wrong note.
**Fix:** The possessive defuses most of it ("*Your* Record" = mine, not theirs-about-me). But it's the strongest argument to test a belonging-led name against the incumbent.

**Attack 3 — "Constellation is gorgeous but overloaded."** The product already uses constellation/RADAR/night-sky for the *matching reveal*. Naming the *whole surface* "My Constellation" may collide with the feature meaning and confuse the IA.
**Fix:** Keep "constellation" as the *kindred/matching* metaphor; don't promote it to the surface name. (Removes it from contention as the top-level name.)

**Attack 4 — "Don't rename in code before market-testing."** Whatever wins, this is a **conversion surface** — the name should be **A/B-tested** (nav label + a one-line hero), not chosen by committee taste. Renaming the route/labels prematurely churns code for an unvalidated guess.
**Fix:** Treat naming as a **test**, not a decision. Ship the spine (Phase 1) with the name behind the same flag, run 2–3 finalists as a label/hero A/B, let return-rate + conversion pick. Keep "My Record" as the internal/code name until data speaks.

**Attack 5 — "App Store discoverability ≠ in-app nav."** The in-app surface name and the *app's* store name/category are different problems; don't conflate them. This panel is about the in-app surface.
**Fix:** Scope locked to the in-app surface; store naming is a separate exercise.

---

## 3. SYNTHESIS — recommendation

**The name is a test, not a verdict — but the panel's recommended finalists to test are:**

1. **Your Record** *(safe upgrade — keeps comprehension, adds second-person warmth; lowest-risk change from incumbent).*
2. **My Place** *(belonging-led, warm, plain-spoken; strongest "I fit here" pull for the wedge segment).*
3. **My Record** *(incumbent control).*

**Locked calls:**
- **Keep the evocative words ("Constellation," "Echoes," "Kindred") for the *sections and headlines inside* the surface — not the surface name.** That's where they're powerful and where comprehension doesn't matter.
- **Don't hard-rename in code yet.** Ship Phase 1's spine with the name as a flagged label + hero line; **A/B the three finalists** on the real metric (D7 return + free→member), with the trust guardrail watched. Internal/route name stays `lab` / "My Record" until a winner emerges.
- The surface's *job* (your experience, held + connected + evolving) matters far more than its label; a great spine with "My Record" beats a mediocre one called "Echoes."

**Recommended next step:** carry "My Record" as the working name into Phase 1; wire the surface title + hero as a single configurable string so the A/B is a one-line change, not a refactor.
