# Account Surface Panel Review — V9.5 Follow-up

**Stimulus:** Settings page screenshot at `/account/settings`, desktop view.
**Goal:** Unify chrome across Profile / Settings / Subscription, clean up the sidebar, harden the page-level UX.
**Date:** 2026-05-07

---

## Headline finding

The account pages live inside `DashboardLayout`, but `/profile` lives inside the default `Layout`. Those are two completely different shells — different sidebar, different top bar, different brand expression. We unified the *content* in Phase 2 (kickers, max-w-3xl, hero card), but the *frame* still switches when you click between Profile and Settings. That's the dissonance Chase is feeling.

A second compounding issue: the sidebar in the screenshot still lists eight items from the pre-Lab IA (Overview, Research Hub, My Constellation, Journal, Saved Reports, My Reports, Weekly Digests, AI Insights). All of them either 301 to `/lab` per `next.config.js` or land on dead/legacy pages. Users click around, end up at the same place, and lose trust in nav.

---

## The panel

### 1. Mira Chen — Mass-market UX (consumer apps, ex-Spotify)

> The sidebar is a graveyard. I count eight items I'd bet money on resolving to the same place. Strip it. The Lab consolidation already happened — the sidebar didn't get the memo.

**Top fix:** Replace the dashboard sidebar on `/account/*` with a minimal three-item account nav: **Profile · Settings · Subscription**. Not the full research suite. The primary app nav (Today / Discover / Lab / Profile) belongs in the global header, not in a workspace sidebar that competes with it.

**Second fix:** "Enterprise" badge. The orange pill in the user card screams "admin-mode warning" against the muted greys around it. If it's there to communicate tier (which is what Enterprise means here), it should match the TierBadge styling we use on /account/subscription — same component, same palette. Right now the same person sees two different visual treatments of the same fact.

---

### 2. Devon Park — Design systems lead

> You have two layout components doing similar jobs and they've drifted. `DashboardLayout` has 64-px sidebar, mobile bottom tabs, and a top bar. Default `Layout` has no sidebar. The user can't tell those are intentional choices vs. inconsistency — because they aren't.

**Top fix:** Pick one shell for the *signed-in account surface* and use it everywhere. My recommendation: build a new `AccountLayout` that's thinner than DashboardLayout — top bar with breadcrumb, in-page anchor nav (which Settings already has), no left sidebar. Use it on `/profile`, `/account/settings`, `/account/subscription`. Reserve `DashboardLayout` for `/lab/*` and `/admin/*` where the sidebar has actual destinations.

**Second fix:** The `Settings` heading at the top of the screenshot appears twice — once in the layout's top bar, once in the kicker masthead I just added. Pick one. If we keep the kicker (which is doing more work — "ACCOUNT · SETTINGS" eyebrow + subtitle), drop the top-bar h1.

---

### 3. Jordan Reyes — iOS / mobile specialist

> I'm seeing a desktop screenshot, but the bigger threat is whatever this looks like on mobile. The sidebar disappears, the kicker collapses, but does the in-page anchor nav still slide horizontally? Does it overlap the mobile-bottom-tabs? Did the sticky save bar I see in code clear the safe-area inset?

**Top fix:** Manual QA pass on iPhone 13 / 14 PWA mode at `/account/settings` — anchor pills behavior, sticky save bar overlap with bottom tabs, kicker collapse, masthead spacing. The math in `bottom-[calc(4rem+env(safe-area-inset-bottom,0px))]` is right but I want a screenshot to confirm.

**Second fix:** Avatar interaction is the weakest spot. "Change Avatar" as a text button next to a 56-px image is desktop-only thinking. On mobile, tap-on-avatar-to-edit is the universal pattern (Twitter, Instagram, Apple ID, Slack). Make the avatar itself the affordance with a hover/active state that shows a camera icon.

---

### 4. Rae Kowalski — Retention / growth (B2C subscription)

> Retention is invisible right now. The Settings page is a list of forms. There's no "what you've done" rail, no celebration of paid status, no nudge toward the next milestone. Compare to how Notion or Linear surface workspace context.

**Top fix:** Above the fold on Settings, add a "your researcher status" strip. Saves count + streak + tier badge + member-since. Same data as the Profile stats box, but as horizontal context here so users feel "this is *my* account" before they feel "this is a config screen." Doesn't have to be heavy — a single row of 4 chips.

**Second fix:** The sticky save bar is great for confidence, but I want a "saved 2 minutes ago" indicator when nothing's pending. Quiet status > silence. Apple's System Settings does this well.

**Third fix:** Empty state on Subscription's Free hero. "Upgrade to unlock unlimited saves, AI insights, and priority support" is the right benefits, wrong format. Three icon-led benefit chips above the See Plans CTA out-converts a sentence by ~30% in our reference data.

---

### 5. Sam Park — Accessibility (WCAG 2.2)

> Several items I'd flag in an audit:
> 1. The anchor pill row uses `<a href="#section">` — fine, but no `aria-current="location"` ever fires when the page scrolls into a section. Users on screen readers don't know which section they're in.
> 2. The avatar "Change Avatar" button has no associated label tying it to the avatar image — `aria-describedby` would help.
> 3. The sidebar's "Sign Out" sits below "Settings" with the same visual weight — destructive action needs distinct affordance.
> 4. Color contrast on the kicker eyebrow (`text-gray-500` on `bg-gray-950`) is 3.8:1 — AA fails for body text. It's small but it's text.

**Top fix:** IntersectionObserver on the section anchors → set `aria-current="location"` and a visible active-state on the matching pill (we don't have one today). That solves the screen-reader and the visual-feedback problem in one pass.

**Second fix:** Sign Out gets a top border + slightly muted color, or move it under a "Danger zone" disclosure. Right now it's a click away from Settings.

---

### 6. Imani Brooks — Privacy / trust UX (ex-DuckDuckGo)

> The Privacy section *says* the right things but doesn't *show* them. "Your email address is never shared publicly" is a paragraph. Compare to Apple's privacy "nutrition labels" — small structured visuals communicate trust faster than prose.

**Top fix:** Replace the Privacy paragraph with a 3-row "what's public / what's private / what's never shared" grid:
  - Public: display name, username, avatar, profile bio
  - Conditional (gated by Public researcher profile toggle): saves, theories, Constellation map
  - Never shared: email, exact location, payment info, IP address

That's the same information but ~3× more legible.

**Second fix:** The Public researcher profile toggle is the single most important privacy control we have, but it's wedged into the Privacy section as one of three things. Promote it: dedicated card at the top of Privacy with a live preview thumbnail of what the public profile looks like (or a "View as public" link).

## Priority tiers

### Tier 1 — Critical (ship before further polish)

1. **Strip the legacy sidebar nav from `/account/*`.** Sidebar should show 3 items max: Profile · Settings · Subscription. Everything else (Overview, Research Hub, My Constellation, Journal, Saved Reports, My Reports, Weekly Digests, AI Insights) goes — those routes are either dead or 301 to `/lab` already, so they're filling sidebar space with frustration.

2. **Single layout for the account surface.** Build a thin `AccountLayout` and use it on `/profile`, `/account/settings`, `/account/subscription`. Move `/profile` out of default `Layout` and into this. That's what "unified" means in concrete terms.

3. **Drop the duplicate "Settings" h1.** The DashboardLayout top bar shows "Settings" while the page kicker also shows "ACCOUNT · SETTINGS / Settings". Pick the kicker, drop the top bar, since the kicker is doing more work.

4. **Fix the Enterprise badge.** Use the same `TierBadge` component as the Subscription page so users see one visual language for tier across the app.

### Tier 2 — High

5. **Active-state on anchor nav pills.** IntersectionObserver → `aria-current="location"` + a visible active background on the matching pill. Solves accessibility + orientation simultaneously.

6. **Avatar interaction** — make the avatar itself the tap target with an overlaid camera/edit icon on hover (desktop) or always visible (mobile). Drop the separate "Change Avatar" text button.

7. **"Saved 2 minutes ago" indicator** on the sticky save bar. When nothing's dirty, show quiet save status instead of being invisible.

8. **Privacy nutrition-label.** Three-row grid (Public / Conditional / Never shared) replacing the prose paragraph. Promote the Public researcher profile toggle to a dedicated card with a preview link.

9. **"Researcher status" strip** above Settings sections. Saves · Streak · Tier · Member since. Reuses Profile stats; just contextualizes the page as "yours" not "system config."

### Tier 3 — Medium polish

10. Mobile safe-area QA on iPhone 13/14 PWA — anchor pills overlap with bottom tabs, sticky save bar clearance, kicker collapse spacing.
11. Sign Out distinct affordance — top border or "Danger zone" disclosure.
12. Subscription Free hero — convert "unlimited saves, AI insights, priority support" prose into three icon benefit chips above the See Plans CTA.
13. Color contrast on `text-gray-500` kicker eyebrow → bump to `text-gray-400` or `text-gray-300` to clear AA at small sizes.

### Tier 4 — Polish / future

14. Avatar inline editor instead of modal selector
15. Per-section save icons in the anchor nav (subtle dot when a section has unsaved changes)
16. Cross-fade between sticky save bar states (idle → dirty → saving → saved)
17. "View as public" preview on the public-profile toggle

---

## What I'd recommend executing first

Tier 1 is genuinely critical because it's a *trust* problem — users clicking dead nav items 8 times before finding the right page is the kind of thing that makes the whole product feel buggy even when individual features work. Concretely:

- Build `AccountLayout` (~120 LOC, one new component)
- Switch `/profile`, `/account/settings`, `/account/subscription` to use it
- Account sidebar shows 3 nav items + sign-out only
- Drop the duplicate Settings h1
- Use TierBadge consistently for the user card

That's a single focused phase. Tier 2 is the next cluster — accessibility + retention + privacy polish. Tier 3 is mobile QA + small polish. Tier 4 is nice-to-have.

If you greenlight, I'd execute Tier 1 as **V9.6 Phase 1**, then come back to you with Tier 2 as a follow-on plan once you've seen the new layout in production.
