# Admin Surface Panel Review — V9.8

**Stimulus:** all 7 admin pages (`/admin`, `report-review`, `media-review`, `ab-testing`, `anchor-cases`, `push-test`, `avatar-review`).
**Brief:** unify chrome, fix mobile + desktop responsiveness, add the new Avatar Review entry to the index.
**Date:** 2026-05-07

---

## What the audit found

- **`/admin` index has 1,360 LOC** — heavy dashboard with stat cards, ingestion jobs, recent activity, multiple grids. 5 hardcoded tile links: report-review, media-review, ab-testing, anchor-cases, push-test. **Avatar-review is missing entirely.**
- **`/admin/media-review` is 1,461 LOC** — biggest page in the codebase. Lots of `<img>` tags, complex tables, multi-step flows.
- **Each page reinvents the masthead.** anchor-cases uses `<a href>` (Next.js anti-pattern); others use `<Link>`. Back-link copy varies (`← Back to Admin`, `Back to admin`). Different `max-w-*` per page (3xl on push-test, 4xl on avatar-review, 7xl on anchor-cases + index).
- **No persistent admin nav.** Every page makes you go back to `/admin` to switch tools.
- **Mobile is mostly untested.** Pages use `grid-cols-1 md:grid-cols-2 lg:grid-cols-4` which is fine for tiles, but no audit on tables, button placement, or sticky bars at small viewports.

## The panel

### 1. Mira Chen — UX / consumer apps (ex-Spotify)

> Admin tools are a different beast from consumer surfaces. The user is YOU and a small handful of operators — not a mass-market visitor. Optimize for fast task switching and information density, not narrative pacing.

**Top fix:** persistent admin sub-nav at the top of every admin page. Pills / tabs across the top: **Overview · Reports · Media · Avatars · Anchors · A/B · Push**. Click any → switch instantly. No more back-then-forward.

**Second fix:** the index is doing too much in one scroll. Stats + ingestion + activity + tile grid + tables. Compress it: hero stat row at top, primary tools as a grid below, defer "recent activity" to a dedicated tab.

---

### 2. Devon Park — Design systems

> Seven pages, seven different mastheads. Build an `AdminLayout` component (thin like AccountLayout from V9.6) that owns the breadcrumb, sub-nav, max-width container, and section-divider styling. Every admin page wraps in it. Mastheads stop drifting.

**Top fix:** new `AdminLayout` component with two slots: `title` and `children`. Auto-renders Paradocs global header → admin sub-nav → page kicker (title + back link) → content area. Single source of truth.

**Second fix:** standardize max-w to `max-w-6xl` for tool pages (ab-testing, anchor-cases, push-test, avatar-review) and `max-w-7xl` for dashboards (index). No more 3xl/4xl outliers.

**Third fix:** unify back-link → "Overview" pill in the sub-nav becomes the "back home" path. Drop the `← Back to Admin` text links. The sub-nav handles wayfinding.

---

### 3. Jordan Reyes — iOS / mobile specialist

> The 1,461-LOC media-review page on a phone is a horror show. Tables overflow horizontally, buttons stack into long columns, sticky bars overlap content. None of this got tested at 375px wide.

**Top fix:** every admin table/list needs an `overflow-x-auto` wrapper at minimum, OR a card-based mobile variant. Card-based is preferred for review queues (avatar-review already does this — the pattern works).

**Second fix:** sub-nav scrolls horizontally on mobile (overflow-x-auto, `whitespace-nowrap`). Same pattern AccountNav uses on `/account/*`.

**Third fix:** action buttons in queues need minimum 44px touch targets per Apple HIG. Avatar-review currently has `px-4 py-1.5` (~30px) — too small for a thumb tap when you're approving 50 avatars.

---

### 4. Rae Kowalski — operator UX / time-to-action

> Admins are operators. They're approving avatars, reviewing reports, sweeping media. The single most-important metric is **time per decision**. Right now, every page reload means re-fetching everything, every back-button means losing context.

**Top fix:** keyboard shortcuts in review queues — `A` for approve, `R` for reject, `J/K` to navigate items. This is what Linear and Superhuman do; admins ship 3x faster.

**Second fix:** auto-refresh on review queues — poll `/api/admin/avatar-queue` every 30s so newly-arrived items appear without a manual refresh.

**Third fix:** pending counts as badges next to each sub-nav pill. "Avatars (3)" tells the operator immediately whether there's work to do without clicking through.

---

### 5. Sam Park — accessibility (WCAG 2.2)

> A few violations across admin pages:
> 1. Many buttons lack `aria-label` when they're icon-only (the X close on push-test, the trash icons in anchor-cases)
> 2. Tab order is broken — admin nav items have no `aria-current="page"`
> 3. `<a>` (instead of `<Link>`) on anchor-cases breaks Next.js's keyboard nav prefetching
> 4. Color contrast on `text-gray-500` for muted body text on `bg-gray-950` is 3.8:1 — fails AA at small sizes

**Top fix:** color contrast pass. `text-gray-500` → `text-gray-400` for any body-size text. Captions/timestamps stay 500.

**Second fix:** every admin nav pill gets `aria-current="page"` when active. Helps screen readers + lets us style active state with `[aria-current="page"]:` in CSS.

**Third fix:** all icon-only buttons get `aria-label` audits. Avatar-review's X close has one; many other admin buttons don't.

---

### 6. Imani Brooks — admin productivity / SaaS tooling

> The bigger meta-issue: admins shouldn't have to memorize a tree of URLs. Stripe Dashboard, Vercel, Linear — all use a single sidebar (or top tab) that's always visible. Paradocs admin currently makes you visit `/admin` between every action.

**Top fix:** sub-nav is non-negotiable. It changes the operator experience from "tool 1 → home → tool 2" to "tool 1 → tool 2." That's a 3x productivity multiplier on a busy day.

**Second fix:** mobile-first layout for sub-nav. Most ad-hoc moderation happens on a phone — a notification arrives, you open the app, you decide. The desktop dashboard view is for batch sessions.

**Third fix:** add a "Mobile-friendly" badge to the homepage as a stake in the ground — make it a literal commitment that admin tools work on mobile, not "should work."

## Synthesized priority tiers

### Tier 1 — Critical (ship first)

1. **Add Avatar Review to `/admin` index.** One missing tile. ~15 LOC.
2. **Build `AdminLayout` component** (~120 LOC). Thin shell with global header → sub-nav strip → page content. Sub-nav: Overview · Reports · Media · Avatars · Anchors · A/B · Push.
3. **Switch all 7 admin pages to use `AdminLayout`.** Drop the per-page mastheads + back-links. Standardize on `max-w-6xl` for tool pages, `max-w-7xl` for the index dashboard.
4. **Mobile sub-nav** — horizontal overflow-scroll same pattern as AccountNav.
5. **`aria-current="page"` on the active sub-nav pill.**

**Estimated scope:** ~250 LOC + edits across 7 files. One commit.

### Tier 2 — High polish

6. **Mobile audit per page** — wrap tables in `overflow-x-auto`, force minimum 44px touch targets on action buttons in queues, fix any fixed-width content.
7. **Color contrast bump** — `text-gray-500` → `text-gray-400` for body-size muted text across admin pages.
8. **Keyboard shortcuts** in review queues (A=approve, R=reject, J/K=navigate). avatar-review first since the queue is shortest, then media-review + report-review.
9. **Pending count badges** on sub-nav pills — "Avatars (3)" when there are items in the queue. Pulls from a single new endpoint that returns counts per queue.

**Estimated scope:** ~350 LOC across 4 files + 1 new endpoint.

### Tier 3 — Medium polish

10. **Auto-refresh on review queues** every 30s (silent poll).
11. **Compress `/admin` index** — collapse "recent activity" into a tab; hero stat row stays at top.
12. **Replace anchor-cases `<a>` with `<Link>`** to fix the Next.js prefetch nag.

### Tier 4 — Future

13. **Toast system for queue actions** — currently approve/reject is silent except for list disappearing. A "Approved Chase's avatar" toast confirms action.
14. **Bulk actions** — checkbox + Approve/Reject Selected on review queues.
15. **Filter + search** on review queues at scale.
